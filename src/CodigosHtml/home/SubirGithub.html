  1. git init
  2. git add .
  3. git commit -m "PrimerCommit "
  4. crear un repositorio en github sin README
  5. crear el repositorio
  6. copiar la dirección https
  7. en la terminal git remote add origin https://github.com/nitram26/laboris
  8. git push --set-upstream origin master

Cómo hacer deploy de tu proyecto con GitHub Pages



Paso a paso:

1. Ve a tu repositorio en GitHub :


2. Haz clic en la pestaña "Settings" (Configuración).

3. En el menú lateral izquierdo, baja hasta la sección "Pages" o "Pages" → "Deployments" .

4. En "Source" (fuente), selecciona:

* Branch: master
* Folder: / (root)

Luego haz clic en Save .

5. GitHub mostrará un mensaje como este:

Your site is ready to be published at:
Proyecto

Copia ese enlace: ¡ese será tu sitio web!

---

Requisitos para que funcione:

* Tu archivo se debe llamar exactamente index.html (lo tienes ✅).
* Todos los archivos deben estar en la raíz del proyecto (no dentro de carpetas como /src o /public ).
* Debe estar en la rama que elegiste como fuente ( master en este caso).

---

¿Y si no aparece todavía?

GitHub Pages a veces demora unos segundos a 1 minuto en activarse. Si visitas el enlace y no carga, espera un poco y vuelve a probar.

Eliminar un Proyecto de Github

A. Eliminar un repositorio completo en GitHub
1. Inicia sesión en GitHub y ve al repositorio que quieres eliminar.
2. Haz clic en “Settings” (configuración) en la barra lateral derecha.
3. Desplázate hasta la sección “Danger Zone” (Zona de peligro).
4. Haz clic en “Delete this repository” (Eliminar este repositorio).
5. Se te pedirá que escribas el nombre del repositorio para confirmar. Escríbelo exactamente y confirma.
6. Si tienes dos factores de autenticación (2FA) activados, se te pedirá ingresar un código de verificación.
7. Confirma la eliminación. El repositorio se eliminará de forma permanente (no se puede deshacer).
